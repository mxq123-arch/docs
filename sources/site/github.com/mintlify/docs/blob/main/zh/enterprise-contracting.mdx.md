法律问题几乎总是首先源于满意度问题。我们相信最好的法务部门就是满意的客户。

这意味着我们尽可能说"是"，行动迅速，只在少数确保平台为每个人正常运行的事项上坚持立场。

本页面说明了这些事项是什么，以及哪些是真正可以协商的。

\## 预期流程

所有 Mintlify Enterprise 协议都会经过法律审查。我们的目标是在三个工作日内完成签署。

我们审查您的修改意见，并将每项与我们的标准立场进行评估。 我们就每个未决事项给出回复。我们带来的是解决方案，而非障碍。 最终文件签署，交易完成。

\## 我们可以灵活处理的内容

采购团队提出的大部分要求，我们都尽力配合。

- 自定义服务级别协议
- 安全认证（Service Organization Control (SOC) 2 Type II、ISO 27001、Digital Operational Resilience Act (DORA)）
- 数据处理协议
- 责任上限
- 赔偿范围
- 适用法律和管辖权
- 保险要求
- 自动续约退出
- 审计权利

\## 六项固定立场

这些是平台运营的结构性基础。我们将逐一解释——不是为了自我辩护，而是因为理解原因通常就能消除顾虑。

\*\*我们的立场：\*\* Mintlify 拥有所有平台代码、功能和方法。 客户拥有其所有内容和数据。

```
**原因：** Mintlify 是一个服务数千客户的多租户平台。如果任何客户可以声称拥有平台功能的所有权，我们就无法向其他人提供这些功能，平台将停止改进。我们希望 Mintlify 随着时间推移越来越好，而不是越来越差。

**这对您意味着什么：** 您的内容、您的数据和您的文档输出完全属于您。我们不对其中任何部分提出权利主张。
```

\*\*我们的立场：\*\* Mintlify 可以使用去标识化的聚合使用数据（功能使用情况、性能指标、搜索模式）用于产品分析和改进。

```
**原因：** 了解功能在整个平台上的表现是我们决定构建什么以及如何优化的依据。没有使用模式，我们就是在盲目飞行——当您依赖我们持续改进时，您也是如此。

**这对您意味着什么：** 我们分析的是聚合信号，而非您的内容。您的具体文档、您用户的数据以及任何可识别信息永远不会被展示或共享。
```

\*\*我们的立场：\*\* 标准上限为 12 个月费用。对于大型交易，我们提高上限。我们不接受无限责任。

```
**原因：** 无上限的风险敞口使我们无法获得保险，这意味着我们必须按最坏情况定价，或者完全拒绝客户关系。两种结果都对您不利。

**这对您意味着什么：** 我们接受年费 2 倍的超级上限，适用于最重要的类别：保密性违规、数据安全事件和知识产权赔偿。我们认为这对高风险类别是公平的，并且有保险支持。
```

\*\*我们的立场：\*\* 客户不得转售或再分发平台访问权限。

```
**原因：** 我们的安全模型、定价和支持承诺建立在与已知客户的直接关系之上。转授权会产生我们无法监控、保护或支持的分发。

**这对您意味着什么：** 您公司家族内的关联方使用——子公司、关联实体、并购场景——完全在标准企业条款的范围内。
```

\*\*我们的立场：\*\* 我们按照 SLA 保证平台正常运行时间。我们按标准"按现状"SaaS 条款提供其他一切。

```
**原因：** 我们使用第三方 LLM 提供商，其输出我们无法保证。没有软件公司能够保证依赖于客户数据、使用场景或第三方集成的结果。

**这对您意味着什么：** 我们将以书面形式保证我们的 LLM 提供商在合同上被禁止将您的数据用于模型训练或改进——这才是真正重要的保护。
```

\*\*我们的立场：\*\* 任何一方均可因重大违约终止协议，并有 30 天的补救期。

```
**原因：** 单方面退出权消除了我们定价的经济基础。

**这对您意味着什么：** 我们的 SLA 结构在服务出现问题的极少数情况下为您提供积分返还。当存在未经补救的实际违约时，可以终止合同。我们不会将客户锁定在不利的处境中——我们只需要双方的共同承诺来证明投资的合理性。
```
