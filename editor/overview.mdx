The Mintlify web editor lets you create, edit, and publish documentation entirely in your browser — no local setup, no command line, and no manual `docs.json` edits required. Every change you make saves automatically, your team can collaborate in real time, and your Git repository stays in sync as the single source of truth.

## How the editor works

**Changes save automatically.** As you type, the editor saves your draft to Mintlify's servers. Your work persists across browser tabs, devices, and network interruptions — so you never lose progress. Saved changes only go live when you explicitly publish them.

**Git stays in sync.** When someone pushes changes to your repository from outside the editor, those changes appear automatically — no pull or page refresh needed. The editor merges non-conflicting changes and highlights anything that needs your attention.

**Publishing writes to Git.** When you publish, the editor commits your changes directly to your repository. On a deployment branch, this updates your live site immediately. On a feature branch, it creates a pull request for review before anything goes live.

**Your team edits together.** Multiple people can edit the same page on the same branch simultaneously. Live cursors show exactly who is editing and where, with collaborator avatars displayed in the toolbar.

## The editor layout

The editor interface has two primary areas:

- **Top bar** — Control the active branch, access the AI agent, open a live preview, and trigger publishing. The top bar also displays collaborator avatars and connection status.
- **Navigation sidebar** — Select any page to open it in the editor, create new pages, reorder content, and manage your overall site structure without editing configuration files directly.
