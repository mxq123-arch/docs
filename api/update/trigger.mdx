Use this endpoint to trigger a rebuild and deployment of your documentation site from the latest commit on your connected Git branch. After calling this endpoint, use the returned `updateId` with [Get Status](/api/update/status) to poll for completion.

## Authentication

This endpoint requires an **Admin API key** (`mint_` prefix) passed as a Bearer token in the `Authorization` header.

## Request

**`POST https://api.mintlify.com/v1/update/trigger`**

### Body Parameters

<ParamField body="deploymentId" type="string" required>
  The unique identifier for your Mintlify deployment. Find this on the
  deployment settings page in your dashboard.
</ParamField>

## Response

<ResponseField name="updateId" type="string">
  A unique identifier for this update. Pass it to `GET /update/{updateId}` to
  poll the build status.
</ResponseField>

<ResponseField name="status" type="string">
  The initial status of the update at the time of the response. Typically
  `pending` immediately after triggering.
</ResponseField>

## Example

<CodeGroup>

```bash cURL
curl https://api.mintlify.com/v1/update/trigger \
  -X POST \
  -H "Authorization: Bearer mint_yourAdminKeyHere" \
  -H "Content-Type: application/json" \
  -d '{"deploymentId": "your-deployment-id"}'
```

```json Response
{
  "updateId": "upd_01hx9k2m3n4p5q6r7s8t9u0v",
  "status": "pending"
}
```

</CodeGroup>
