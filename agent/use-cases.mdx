The agent is flexible enough to fit into many different documentation workflows. The examples below show how teams use the agent to capture knowledge, ship accurate docs, and reduce manual maintenance work. Start with the approach that fits how your team already works, then adapt it to your specific needs.

<Accordion title="Iterate on a prompt in a Slack thread">
  Prompt the agent in Slack, then continue mentioning it with `@mintlify` in the same thread to refine and iterate on the pull request it creates. This lets you have a conversation with the agent until the output is exactly right — without leaving Slack.

  **Example flow:**

  1. `@mintlify Our quickstart page needs a new section on inviting collaborators`
  2. `@mintlify The new section should be called "Inviting collaborators"`
  3. `@mintlify Add a note that collaborators must have a verified email address`

  Each message refines the same pull request rather than creating a new one.
</Accordion>

<Accordion title="Start with the agent, finish manually">
  Prompt the agent to begin a task, then check out the branch it creates and finish the work in your local environment or the Mintlify web editor. This is useful when you want a head start on a larger project or need to add details that require local context.

  **Example:** `@mintlify Update the quickstart page to include information about inviting collaborators`

  After the agent opens a PR, check out its branch and make any additional changes using your preferred editor. Merge when the page is ready.
</Accordion>

<Accordion title="Update docs when merging feature changes">
  When you merge a feature pull request, share the PR link with the agent so it can update the relevant documentation pages to reflect the changes. This keeps docs in sync with the product without a separate documentation sprint.

  **Example:** `@mintlify This PR adds a new authentication method. Update the docs to include the new auth flow: [PR link]`
</Accordion>

<Accordion title="Generate release notes from a pull request">
  Prompt the agent with a pull request link to generate release notes or changelog updates based on the commit history and PR description. This saves time when preparing release communications and ensures the notes reflect what actually changed.

  **Example:** `@mintlify Generate release notes for this PR: [PR link]`
</Accordion>

<Accordion title="Address pull request review feedback">
  Share a documentation pull request link with the agent to resolve reviewer comments. The agent reads the review feedback and makes targeted edits to address each comment, then updates the existing PR.

  **Example:** `@mintlify Address the review comments on this PR: [PR link]`
</Accordion>

<Accordion title="Generate code examples">
  Ask the agent to write or improve code examples for features throughout your docs. The agent can generate examples for a specific page or search across your entire documentation to add examples wherever they're missing.

  **Example:** `@mintlify Generate a code example to make the authentication method easier to understand`

  If you have an [AGENTS.md](/agent/customize) file that specifies your preferred programming language or coding conventions, the agent uses those preferences automatically.
</Accordion>

<Accordion title="Add images or files to your docs">
  Attach files or images directly to your Slack message when prompting the agent. The agent processes the attachment and incorporates it into the documentation at the appropriate location.

  **Example:** Attach a diagram image and send `@mintlify Add this diagram to the architecture overview page`

  The agent handles uploading the file, adding it to the correct page, and writing descriptive alt text.
</Accordion>

<Accordion title="Review existing content">
  Ask the agent to review existing content for technical accuracy, style consistency, grammar, or other issues. This is useful after product changes that might have made parts of your documentation stale.

  **Example:** `@mintlify Review the API rate limiting section. We changed limits last month`

  The agent opens a pull request with its suggested corrections so your team can review and approve each change.
</Accordion>

<Accordion title="Respond to user feedback">
  Share feedback you've received from users with the agent to make targeted improvements to your docs. The agent can analyze the feedback, identify what might be confusing, and propose edits to address the issue.

  **Example:** `@mintlify Users are getting confused by step 3 in the setup guide. What might be making it unclear?`
</Accordion>

<Accordion title="Automate with automations">
  Create automations to run the agent automatically on a schedule or when a push event occurs in a connected repository. Automations are ideal for recurring maintenance tasks that don't need manual intervention — things like writing changelogs, syncing docs with code changes, checking for broken links, or running style audits.

  See [Automations overview](/automations/overview) for setup instructions and the full list of automation types.
</Accordion>

<Accordion title="Automate with the API">
  Integrate the agent into your existing CI/CD or automation tooling using the agent API endpoints. You can [create jobs](/api/agent/v2/create-agent-job), [get job status](/api/agent/v2/get-agent-job), and [send messages](/api/agent/v2/send-message) programmatically — enabling documentation updates to happen automatically whenever code changes occur.

  When creating jobs via the API, use the `asDraft` parameter to control whether the agent opens a draft PR or a ready-to-merge PR:

  ```json
  {
    "prompt": "Update the changelog for the v2.3 release",
    "asDraft": true
  }
  ```

  `asDraft` defaults to `false`, so pull requests are ready for immediate review and merging by default. Set `asDraft: true` when you want to inspect the output before it's visible to reviewers.
</Accordion>
