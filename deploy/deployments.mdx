Your documentation site automatically deploys whenever you push changes to your connected repository. This requires the Mintlify GitHub App to be properly installed and connected to your repository. Each successful push triggers a new build, and your live site updates once the build completes.

<Tip>
  If your latest changes aren't appearing on your live site, first check that the GitHub account or organization owning your docs repository has the GitHub App installed. See [GitHub troubleshooting](/deploy/github#troubleshooting) for details.
</Tip>

## Manually trigger a deployment

If you have the GitHub App installed but changes still aren't deploying, you can manually trigger a build from your dashboard.

1. Confirm that your latest commit appears in your docs repository and did not encounter any errors.
2. Go to your [dashboard](https://app.mintlify.com) and click the **Deploy** button to start a new build.

Manually triggering a deployment pulls the latest commit from your configured branch and rebuilds your documentation site.

## Delete a deployment

You can permanently delete a deployment from the [Danger zone](https://app.mintlify.com/settings/organization/danger-zone) section of your dashboard settings.

<Warning>
  Deleting a deployment is irreversible and removes all deployment data, including any associated preview deployments. If you have an active subscription, you receive a prorated credit for any unused time remaining in your billing period.
</Warning>

To delete a deployment:

1. Go to the [Danger zone](https://app.mintlify.com/settings/organization/danger-zone) in your dashboard settings.
2. In the **Delete my deployment** section, provide a reason for deletion.
3. Click the delete button and confirm that you want to proceed.

If you have multiple deployments, Mintlify redirects you to another deployment after deletion. If you delete your only deployment, Mintlify redirects you to the Mintlify homepage. The organization admin receives an email notification confirming the deletion.
